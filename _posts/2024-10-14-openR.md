### 论文概述

这篇技术报告介绍了 OpenR，这是一个旨在增强大型语言模型 (LLM) 推理能力的开源框架。OpenR 将数据获取、强化学习训练（包括在线和离线）以及非自回归解码统一到一个软件平台中。其目标是建立一个开源平台和社区，以加速 LLM 推理的发展。受 OpenAI 的 o1 模型的启发，该模型通过逐步推理和强化学习展示了改进的推理能力，OpenR 集成了测试时计算、强化学习和过程监督来改进 LLM 中的推理。

### 核心思想

OpenR 的核心思想是将 **过程监督**、**强化学习 (RL)** 和 **测试时计算策略**（例如引导搜索）相结合，以提高 LLM 的推理能力。它不再仅仅关注预训练期间扩展模型参数，而是利用更智能的推理策略在测试时优化模型输出。这些技术帮助模型逐步完善其推理，使其能够在测试时计算过程中暂停、评估中间推理并选择更好的解决方案路径。

### 关键创新

1. **首个探索 OpenAI o1 模型核心方法的开源框架**: OpenR 利用强化学习技术，实现了超越传统自回归方法的先进推理能力。
2. **整合过程监督、强化学习和测试时计算**: 通过将这三种方法结合，OpenR 显著提升了 LLM 的推理性能。
3. **提供开源平台**: OpenR 包括代码、模型和数据集，促进 LLM 推理研究的协作与发展，加速该领域的进步。

### 算法流程

OpenR 的算法流程可以概括为以下几个阶段:

1. **数据增强**: 为过程监督收集数据，提供逐步反馈，而不是仅仅依赖最终答案。例如，利用 OmegaPRM 方法自动生成合成样本，构建 MATH-APS 数据集。
    - **举例**: 对于一个数学问题，OpenR 不仅收集最终答案，还会收集模型生成每一步推理的中间结果，并为每一步打上“正确”或“错误”的标签，用于训练过程奖励模型。
2. **过程奖励模型 (PRM) 的监督训练**: 利用收集到的过程监督数据，通过监督学习训练 PRM，使其能够评估推理过程中每一步的质量。
    - **举例**: 将数学问题和模型生成的推理步骤输入到 LLM 中，训练 LLM 在每一步推理结束后预测一个“+”或“-”的标签，代表该步骤的正确性。
3. **策略学习**: 将数学问题转化为语言增强的马尔可夫决策过程 (MDP)，利用强化学习算法（如 PPO 或 GRPO）训练 LLM 策略，使其能够根据 PRM 的反馈生成更准确、更有意义的推理过程。
    - **举例**: 将数学问题作为初始状态输入到 LLM 中，LLM 生成一个推理步骤作为动作，PRM 评估该动作的质量并给出奖励，LLM 根据奖励更新策略，重复此过程直到找到最终答案。
4. **解码**: 在测试时，使用引导搜索和规划方法，例如 Beam Search 或 Best-of-N，结合 PRM 对生成的推理步骤进行评估，选择最佳答案。
    - **举例**: 使用 Beam Search 生成多个推理路径，利用 PRM 对每条路径的每一步进行评分，选择得分最高的路径作为最终答案。

### 实验结果

OpenR 在 MATH 数据集上进行了评估，结果表明过程奖励模型和引导搜索的结合将测试时推理性能提高了约 10%。实验还证明了 OpenR 框架在优化 LLM 推理方面的有效性，特别是在测试时计算和强化学习方面。

### 贡献

OpenR 的主要贡献在于：

1. 提供了一个开源框架，用于研究和改进 LLM 的推理能力。
2. 展示了将过程监督、强化学习和测试时计算结合起来可以显著提高 LLM 的推理性能。
3. 提供了用于训练和评估 LLM 推理能力的工具和资源，包括数据集、模型和代码。

### 局限性

1. **实验规模有限**: 由于计算资源的限制，实验主要在较小规模的模型和数据集上进行。
2. **过程监督数据有限**: 尽管使用了 PRM800K 和 MATH-APS 等数据集，但过程监督数据的规模和多样性仍然有限。
